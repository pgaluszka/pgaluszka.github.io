GitHub Gist – po co i na co to?

Zdarzało się Wam, że potrzebowaliście wrzucić gdzieś jakiś fragment tekstu/kodu tak, aby się nim podzielić, komuś wysłać lub podzielić się jakimś fragmentem słowa pisanego? Przynajmniej raz w życiu każdemu się coś takiego przytrafiło. Może to być plik tekstowy, plik z kodem lub też wysłany link z Pastebin. Skoro już wspomniałem Pastebin to należy też opowiedzieć o tym czym jest ta usługa oraz o tym czym jest GitHub Gist. Obie usługi są do siebie bardzo podobne i warto je znać z kilku powodów. Zapytacie dlaczego? Już śpieszę z wyjaśnieniem o co w ogóle chodzi. 

O co w zasadzie chodzi?

GitHub Gist jest bardzo podobny do starego, dobrego Pastebin, przy czym ma trochę więcej możliwości. Można stwierdzić, że podstawowa funkcjonalność jest wręcz identyczna. Zanim jednak o tej funkcjonalności i pozostałych funkcjach opowiem i pokażę przykład działania tego systemu to wcześniej opiszę Wam czym jest Pastebin.
Pastebin – aplikacja internetowa, umożliwiająca wklejanie tekstu i udostępnienie go innym po podaniu odpowiedniego linku. Usługa taka jest szczególnie przydatna, gdy konieczne jest przesłanie dłuższego tekstu czy też kodu źródłowego np. poprzez komunikator internetowy, czy też kanał IRC. Niektóre z nich obsługują kolorowanie składni dla wybranych języków programowania czy też dodawanie hiperłączy do dokumentacji z opisem konkretnego polecenia.
~https://pl.wikipedia.org/wiki/Pastebin

No dobra, skoro już wiemy o co chodzi to przejdźmy do sedna…

Idąc za przykładem opisanej powyżej aplikacji, którą, de facto, poznałem jako pierwszą, do dzielenia się kodem źródłowym to rozpocznę od tej właśnie funkcjonalności GitHub Gista.
O samej możliwości współdzielenia kodu nie ma co za wiele się rozpisywać – wiadomo o co chodzi przecież. Jednakże w przy tym aspekcie chciałbym zaznaczyć jeszcze tylko jedną rzecz, która przemawia za używaniem Gist’a. Już w 2014 roku GitHub był repozytorium dla 12 milionów projektów. Mówcie co chcecie, ale jest to naprawdę sporo. Jako, że w momencie pisania tego wpisu mamy koniec 2016 roku to możecie sobie wyobrazić jak duża jest teraz ta liczba.
1. Pisać możesz anonimowo.
Jeżeli chcesz korzystać z usługi GitHub Gist to wcale nie musisz posiadać konta GitHub. Aby móc korzystać z usługi Gist wystarczy, że wejdziecie na adres http://gist.github.com/. Następne co trzeba wykonać to wpisać jakiś blok tekstu w naszego Gist’a i wszystko gotowe. Co najważniejsze, możecie ustawić prywatność Gist’a jako całkowicie prywatny – utworzone fragmenty teksty bądź kodu źródłowego będą dostępne tylko dla osób, które otrzymają linka od osoby, która utworzyła dany wpis.
2. Śledź zmiany wprowadzone w utworzonym Gist’cie.
Druga, bardzo ważna i w sumie przydatna funkcja GitHub Gist’a to śledzenie zmian, które zostały wprowadzne w już opublikowanych materiałach. Wystarczy, że użytkownik kliknie w „REVISIONS” i już może śledzić zmiany wprowadzane w konkrtenym Gist’cie.
3. Publikuj tekst jako „Rich text”.
Jak już pewnie zauważyliście, GitHub Gist jest w stanie przełknąć tylko tekst. Żeby wyglądało to jakoś ładnie oraz nie miało poprawne formatowanie, można użyć formatowania Markdown i dzięki temu opublikowany tekst wygląda znacznie ładniej. Każdy kto wie czym jest Markdown wie, że pisanie z użyciem tej składnie jest bardzo przyjemne, a efekt jest bardzo zadowalający. Tak na marginesie pisząc, ten post jest napisany w całości przy użyciu Markdown i bezpośrednio na githubie bez użycia żadnego pakiety biurowego. Trzeba pamiętać o tym, żeby na samym końcu zapisać tworzony plik z rozszerzeniem *.md.
4. Gist’a możesz użyć jako platformy do umieszczania treści.
Jak zdajecie sobie z tego w zupełności sprawę, istnieje wiele silników/serwisów do tworzenia właśnych treści w internecie. Prawdopodbnie znacie Blogger’a czy Tumblr’a. Na pewno nie zdajecie sobie sprawy z tego, że możecie używać GitHub Gist jak platformy do umieszczania treści w internecie. Na dobrą sprawę nie tylko Gist ma taką funkcjonalność, ale również pozostałe funkcjonalności GitHub’a razem wzięte mogą służyć za taki generator.
5. Hostuj pojedynczą stronę internetową na GitHub Gist
Jako, że już wspomniałem o tym, że GitHub Gist może służyć jako platforma do umieszczania treści to w sumie ten punkt rozumie się sam przez siebie. Przecież jeżeli można hostować pojedynczą stronę internetową za pomocą tej usługi to dlaczego nie stworzyć sobie z takich Gist’ów całej strony internowej? Przecież nawet oczywistym jest, że można pisać swój kod w HTML, CSS i JavaScript. Jedyne co trzeba zrobić to efekt tej pracy zapisać za jako index.html. Po takiej operacji należy użyć bl.ocks.org i stronka stoi. Przy czym wydaje mi się, że lepszym rozwiązaniem może być pisanie prostej strony w Markdown i poprostu umieszczenie jej na GitHub’ie. Wiem, że takie strony nie będą zachwycać, ale w sumie od czego mamy GitHub Pages?
6. Zrób Pan sobie task listę!
Może to zabrzmi cholernie śmiesznie, ale Gist może nawet Wam posłużyć jako lista zadań. Wcale nie to nie musi być trudne, gdyż wygląda to tak jakbyście zrobili to notatniku.
Może się to przydać o tyle, że jeżeli zaczynacie jakiś projekt z ograniczonymi zasobami to możecie wykorzytać tekie Gist’a przy pracy nad wspomnianym wcześniej projektem. Takiego niepublicznego gista możemy używać jako task listy.
W sumie to wiecie na czym polega task lista to co ja Wam będę truł… Ogarnijcie sobie tylko ten przykład poniżej.

- [x] Zeszkalować kogoś
- [ ] Zadzwonić do pomocy technicznej
- [x] Wznowić subskrypcję Office 365
- [ ] Pizza na obiad?

7. Używaj Gist jako narzędzia typu Web Clipper
Istnieje sobie taka mała wtyczka/add-on dla Google Chrome, która się nazywa Gist-Box (link) i dzięki niej możecie używać usługi GitHub Gist jako takiego Web Clippera.
Zapiski z internetów może zapisywać jako publiczne i prywatne Gist’y. Możecie dodawać do takich materiałów etykiety oraz hashtagi, aby łatwiej znajdować je w przyszłości.
8. Możesz umieścić sobie wcześniej utworzonego Gista na stronie internetowej
Jako, że GitHub jest platformą bardzo mocno programistyczną to w sumie fajnie by było naszego jakiegoś Gist’a umieścić na stronie internetowej lub blogu, prawda?
Nic prostszego… wystarczy tylko umieścić taki kawałek kodu:

<script src="https://gist.github.com/username/gist-id.js"></script>

Najlepsze jest to, że ten mały kawałek JavaScript’u odnosi się bezpośrednio do kawałku tekstu, którego ID został umieszczony w podanym powyżej skrypcie.
Jeżli go sobie zmienicie już na GitHub’ie to zmieni się on również na Waszej stronie internetowej. Dzięki wklejaniu Waszego kodu w taki sposób, ktoś może sklonować Wasze rozwiązanie i je rozwijać lub nawet zrobić forka (Hail GitHub!).
Jeżeli ktoś z Was prowadzi jakąś stronę przy użyciu WordPress’a to istnieje również plugin (link) oraz shortcode (link), który możecie wykorzystać.

Podsumowanie posta

Mam nadzieję, że podane powyżej argumenty Wam w jakiś sposób przybliżą GitHub Gist’a.
Mnie się strasznie podoba to rozwiązanie, gdyż w bardzo łatwy sposób można zarządzać tekstem, którego wielu z Was tworzy bardzo dużo codziennie. Jeżeli jeszcze mielibyście ochotę połączyć tę funkcjonalność portalu GitHub z Markdown, o którym kilka razy w tym wpisie wspominałem to możecie utworzyć całkiem potężną maszynę blogową.
